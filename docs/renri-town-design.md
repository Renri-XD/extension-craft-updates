# Renri's Town 設計書

Renri'sシリーズの旗艦MOD。「自分の手で建てた街に、人が住み、店がまわり、観光客が訪れる」。

既存のRenri's Diner / Inn / Market / Slot / Quest Board / Bestiary / Jukebox を
「街の施設」として束ねるハブMODであり、共通コイン経済の最終形。

---

## 1. ポジショニング — MineColoniesとの差別化

Renri's Town はコロニー管理シムではない。**「経営 × まちづくり × 人間関係」**であり、
比較対象は MineColonies ではなく Stardew Valley / どうぶつの森 / Dinkum。

| 軸 | MineColonies | Renri's Town |
|---|---|---|
| 建築 | NPCが設計図から建てる。プレイヤーは資材係になりがち | **プレイヤーが建てた建物に住民が住む。建築そのものがゲームプレイ** |
| 経済の向き | 内向き（自給自足の生産チェーン） | **外向き（観光客が街にコインを落とす商業）** |
| NPC | 職業ユニット＝労働力 | **個性ある隣人（好物・友好度・個人依頼・雇用契約）** |
| 進行 | 建物ティア＋研究ツリー | **美観・評判・街ランク** |
| プレイ感 | 重量級の管理シム（要・長時間プレイ） | **コージー経営（1日10分でも1周まわる）** |
| 建築様式 | スキマティック準拠。自由建築の認定は手間 | **どんな建築でもスコア化される（和風・洋風・洞窟街もOK）** |
| エコシステム | 単体で完結 | **Renri's全MODのハブ。シリーズが増えるほど街の施設が増える** |

差別化の核は2点：
1. **建築の主体がプレイヤー**であること。MineColoniesは「NPCの街を眺めるMOD」、
   Renri's Townは「自分の街が生き始めるMOD」。
2. **経済が外向き**であること。MineColoniesの目的は自給自足だが、
   Renri's Townの目的は「客を呼んで稼ぐ」。既存4店舗MODがそのまま収益施設になる。

## 2. 注目を集める3つのフック（バズる瞬間）

CreateにおけるMovingContraption（動く建築物）に相当する、
「クリップ1本で伝わる見せ場」を最初から設計に入れる。

### フック① 街が生きて見える瞬間（1.0の見せ場）
- 夕方になると住民が家路につき、家の窓に灯りがともり、店ののれんが下がる。
- 朝は店が開き、住民が通勤し、ベンチで談笑する。
- 実装的には住民のデイリースケジュールAI＋「営業時間」ステートの見た目連動だけで成立する。
  **費用対効果が最も高い演出**。スクリーンショット・動画映えの土台。

### フック② 自分の店が自走する瞬間（1.1の見せ場）
- 住民を雇うと、Dinerで配膳し、Innでチェックイン対応し、Marketで呼び込みをする。
- 「手動で回していた自分の食堂を、雇ったNPCが回している」動画はSNSで最も拡散しやすい。
- 既存MODユーザー全員に直接刺さるアップデートでもある。

### フック③ 観光客のレビューと絵ハガキ（1.2の見せ場）
- 観光客は帰り際に星付きレビューを残す（テンプレート＋店名・料理名・体験の合成で
  手続き生成。「★★★★☆ 『月見亭』のシチューが忘れられない。ベッドは少し硬かった」）。
- レビュー掲示板ブロックに蓄積され、街の評判スコアに反映される。
- 美観スコアの高い地点では観光客が写真を撮り、「絵ハガキ」アイテムを落とすことがある。
  **プレイヤーの建築が作品として承認される装置**であり、共有したくなる最大の動機になる。
  - 理想実装：観光客視点のレンダリングをテクスチャ化した1点物アイテム（Exposure系の技術）。
  - フォールバック：地点名＋美観スコア＋日付入りの意匠アイテムでも成立する。

## 3. 飽きさせない設計 — 3層ループ＋ランダムイベント

リテンションは「短・中・長のループが常に1つ以上未完了である」ことで作る。

### 日次ループ（5〜10分で1周）
1. 開店準備（仕込み・品出し）
2. 客と観光客の対応（雇用が進むほど自動化）
3. 夕方に帳簿を確認 — **その日の売上・客数・人気メニューが数字とグラフで出る**
4. 住民1人と会話 or プレゼント（友好度は1日1回制で「明日も話そう」を作る）

「数字が伸びていくのを毎日確認する」快感が日次の核。帳簿GUIには前日比を必ず出す。

### 週次ループ
- 祭りの準備（食材・装飾の納品クエスト）と開催
- 新住民の勧誘（Innに泊まった旅人を口説く）
- 施設を1つ増築 → 美観と評判が目に見えて動く

### 長期ループ
- 街ランク：集落 → 村 → 町 → 都市 → 王国認定（認定書が届く演出）
- 住民アルバムの完成（Bestiaryと同じUX言語で「住民図鑑」）
- レビュー平均★4.5、絵ハガキコレクション、祭りトロフィー

### ランダムイベント（マンネリの直接対策）
毎朝、低確率で1つ発生。プレイヤーの「今日やること」を外から供給する。
- 行商人が珍しい装飾ブロックを売りに来る
- グルメ評論家の抜き打ち来店（高評価なら数日間、観光客ブースト）
- 雨の日：Innの宿泊需要アップ／露店の売上ダウン（施設間で明暗が出る）
- 住民の個人依頼（好物を作ってほしい、家具を置いてほしい 等 — Quest Board連携）
- 迷子の子供、住民同士の記念日 などの小さな物語イベント

## 4. システム詳細

### 4.1 役場と街範囲
- **役場（Town Hall）**ブロックを設置して街を登録。街範囲は役場中心の半径（ランク依存で拡大）。
- 役場GUI：街名設定、街ランク、人口、評判、美観、帳簿タブ、祭りタブ。

### 4.2 住宅判定と快適度
- 「ベッド＋ドア＋屋根で囲われた空間」を住宅として検出（Innの客室判定ロジックを再利用）。
- 快適度スコア：広さ、光源、家具の数と種類（Diner装飾ショップ家具はボーナス）。
- 快適度が高い家ほど良い住民（レア性格・高スキル）が入居する。

### 4.3 住民
- 入居経路：Innに泊まった旅人へ「移住の勧誘」／街の評判が高いと自発的に移住希望者が現れる。
- パラメータ：名前、性格（のんびり・働き者・グルメ・夜型…）、好物、得意職、友好度（ハート0〜10）。
- スケジュールAI：起床 → 仕事 or 散策 → 店に立ち寄る（**住民自身もコインを落とす**）→ 帰宅 → 就寝。
- 友好度：会話1日1回、プレゼント（好物で大きく上昇）、依頼達成。
  ハート解禁で小さな個人イベントと専用アイテムの贈り物。

### 4.4 雇用（既存MOD統合の本丸）
- 役場 or 店舗の雇用契約書で住民を配属。
- **Diner**：ウェイター（完成料理をカウンターから客席へ配膳）、後続でコック（登録レシピを材料があれば調理）。
- **Inn**：フロント係（チェックイン／アウトと集金を自動化）。
- **Market**：売り子（販売速度アップ＋呼び込みで来客増）。
- **Slot**：ディーラー（テーブルの無人営業＋カジノの集客）。
- 給料は日給制でコイン。帳簿から自動天引き。未払いだと「休憩中」になって働かない
  （ペナルティもコージーに保つ。死亡・反乱はさせない）。

### 4.5 美観スコア（建築を報酬化するコアアルゴリズム）
- 街範囲を16×16のセルに分割し、セルごとに算出 → 合計が街の美観。
- 加点対象はブロックタグ `#renris_town/decor` で管理（花・ランタン・のれん・提灯・
  道ブロック・柵・旗・Renri装飾家具…）。**データパックで拡張可能にして
  他MODの装飾ブロックも登録できるようにする**（Createのアドオン文化と同じ拡張性）。
- スパム対策：同一ブロックは平方根スケーリングで逓減。種類の多様性にボーナス。
- 道の接続率・街灯の被覆率も加点。負タグ（土くれ・ゴミ系）は減点。
- **「街づくりの眼鏡」**アイテム：装着すると美観ヒートマップをオーバーレイ表示。
  Createのゴーグルに相当する「システムが見える化される快感」を担う。

### 4.6 観光客
- 到着点：街道の入口ゲート、桟橋、（Create等の）駅に隣接した観光案内所ブロック。
- 街の評判×美観×施設の種類で、1日の観光客数と客単価が決まる。
- 行動AI：入口 → 美観の高いセルを巡回 → 店で消費 → レビューを残して退場。
- 消費は既存店舗の客システムにそのまま接続（Dinerの客・Innの旅人として振る舞う）。

### 4.7 街ランクと解禁表
| ランク | 条件（例） | 解禁 |
|---|---|---|
| 集落 | 役場設置 | 住民4人、基本装飾 |
| 村 | 人口4＋施設2種 | 雇用、住民8人、街灯・道具ブロック |
| 町 | 人口8＋美観一定＋評判★3 | 観光案内所、祭り（小）、住民16人 |
| 都市 | 人口16＋施設5種＋評判★4 | 大祭り、貿易契約、住民24人 |
| 王国認定 | 人口24＋評判★4.5＋祭り実績 | 認定書、特別住民、金の街灯 等の栄誉装飾 |

### 4.8 祭り
- 祭り広場（やぐら／ステージの小型マルチブロック）＋開催コスト（コイン＋納品クエスト）。
- 開催中：住民と観光客が広場に集まり、屋台にDinerの料理が並び、花火、Jukeboxの音楽、
  カジノナイト（Slot連携）。
- 結果：評判ブースト（翌週の観光客増）、トロフィー、住民の友好度全体上昇。

### 4.9 帳簿・経済
- 役場の帳簿タブ：全店舗の売上・人件費・祭り収支を日次で集計。前日比と週間グラフ。
- 財布（Slot 1.2.0）を口座として扱い、給料・税・祭り費用を自動精算。
- 貿易契約（後期）：「小麦512を3日以内に納品 → まとまったコイン」のような
  輸出コンテナ。Marketの上位互換として大量生産の出口を作る。

## 5. 技術メモ

- **再利用できる既存実装**：DinerのカスタムNPC描画（7色衣装・頭バグ修正済みの知見）、
  Innの旅人スポーン／ベッド判定／チェックアウト集金、Marketのオフライン売上シミュレーション、
  BestiaryのコレクションUI、共通コイン＋財布API。ゼロから作るのは
  スケジュールAI・美観スキャナ・帳簿集計の3つが主。
- **パフォーマンス方針**：
  - 住民数はランク上限でキャップ（4/8/16/24/32）。
  - 遠距離の住民はLOD AI（パスファインディング停止、位置テレポート補間）。
  - 美観スキャンは常時計算しない。ブロック更新のあったセルだけ再計算＋眼鏡使用時に強制更新。
  - 非ロード時はMarket方式のオフラインシミュレーション（帰ってきたら帳簿にまとめて反映）。
- **マルチプレイ**：街は1つを共同経営（役場に共同経営者登録）。
  将来的に「隣町」を建てて観光客を取り合う対抗戦も可能な設計にしておく。

## 6. ロードマップ（シリーズ流の段階リリース）

各リリースに必ず「changelogの1行目になる見せ場」を置く。

| Ver | テーマ | 見せ場（フック） |
|---|---|---|
| 1.0 | 役場・移住・友好度 | 「夕方、あなたの街に灯りがともる」 |
| 1.1 | 雇用 | 「あなたの食堂を、住民が回してくれる」 |
| 1.2 | 観光客・美観・レビュー | 「観光客があなたの街をレビューする。絵ハガキも」 |
| 1.3 | 祭り | 「街の祭りを開こう。屋台・花火・カジノナイト」 |
| 1.4 | 帳簿・税・貿易 | 「街まるごとの帳簿と、王都との貿易契約」 |

1.0の時点で「住民が住む・話せる・街が生きて見える」まで届いていることが重要。
雇用や観光を待たずとも、フック①だけで既存パックの体験が一段変わる。
